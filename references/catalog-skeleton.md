# SKILLS-CATALOG 骨架模板

> 用于在任意 agent host 目录中新建或更新 SKILLS-CATALOG.md。
> 已有内容的 host 按现有结构动态更新；空目录按此骨架初始化。

---

## 模板

```markdown
# [Host 名] Skills 技能目录

> **最后更新**: YYYY-MM-DD
> **总计**: N 技能 (本地 + 外部链接)
> **调用方式**: `/skill-name` 或通过 Skill tool 调用

---

## 目录

1. [分类名 A](#1-分类名-a)
2. [分类名 B](#2-分类名-b)
...
N. [技能来源溯源表](#n-技能来源溯源表)

---

## 技能来源总览

| 阵营 | 作者/组织 | 技能数 | 许可证 | 说明 |
|------|-----------|--------|--------|------|
| **阵营名** | [作者](GitHub URL) | N | MIT / Apache 2.0 | 一句话说明 |

---

## 1. 分类名 A

简要描述该分类覆盖的能力域。

| 技能 | 用途 | 来源 | 触发场景 |
|------|------|------|----------|
| `skill-name` | 一句话用途 | [作者](GitHub URL) | 触发关键词/场景 |

---

## N. 技能来源溯源表

按作者/组织分组的完整溯源。

### 阵营名 (N 技能)

许可证：MIT | 来源：[repo](GitHub URL)

```
skill-a, skill-b, skill-c
```

### 社区独立作者

| 技能 | 作者 | 来源 |
|------|------|------|
| `skill-name` | **author** | [GitHub URL](URL) |

### 来源未明

```
unknown-skill
```

### 用户自建 (N 技能)

以下技能经三级搜索（GitHub / Marketplace / find-skills）排查后未找到公开来源，归类为用户自建。

```
custom-skill-a, custom-skill-b
```

> 最后排查时间：YYYY-MM-DD

## 标记说明

| 标记 | 含义 |
|------|------|
| (无标记) | 本地技能 |
| 🔗 | 外部链接 |
| `prefix:` | 技能包前缀调用 |
| `自建` | 经溯源排查确认为用户自建 |

## 快速选择指南

```
需要做什么？
│
├─ 场景 A → /skill-1 → /skill-2
├─ 场景 B → /skill-3
└─ 找新技能 → /find-skills
```

---

> **维护说明**: 每次添加或删除技能时，更新对应分类表格、溯源表和总计数字。
```

---

## 设计约束

| 维度 | 约束 |
|------|------|
| 表格格式 | 统一四列：技能 \| 用途 \| 来源 \| 触发场景 |
| 来源列 | 必须带 GitHub 链接（有官网的补官网） |
| 溯源表 | 按作者分组，技能列表用 code block |
| 快速选择 | 决策树格式，场景 → 技能链 |
| 维护仪式 | 增删技能 → 更新表格 + 溯源 + 总计 |

## 分类策略

不硬编码分类名。按以下优先级决定分类归属：

1. **已有分类匹配** — 读现有章节标题，按技能 description 关键词匹配最近章节
2. **新建分类** — 10+ 技能属于同一领域且无现有匹配时，新建独立章节
3. **兜底** — 少量杂散技能放入「工具与基础设施」
